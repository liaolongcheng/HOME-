RENT-API

api for the rent app

Usage

1. districts

load all districts data

method: REST (GET)
url: /areas/:lang/districts.:format
variables:

  • lang: language of returned (required, en|zh)
  • format: format of returned (required, only support json currently)

response:

[
  "Xuhui",
  "Changning",
  "Pudong",
  "Luwan",
  "Jing'an",
  "Huangpu",
  "Hongkou"
]

2. CBD

query CBDs by district

method: REST (GET)
url: /areas/:lang/:district/CBDs.:format
variables:

  • lang: language of returned (required, en|zh)
  • district: which district you want to query(required, such as Xuhui or 徐汇区)
  • format: format of returned (required, only support json currently)

5. publish house

publish the house information

method: REST (POST)
url: /houses/publish.:format
variables:

  • format: format of returned (required, only support json currently)
  • name_zh: name in Chinese
  • name_en: name in English (required)
  • houseType: type of the house(a reference id of houseType model)
  • floor: floor of the house
  • longitude
  • latitude
  • price
  • rooms
  • share
  • size
  • totalFloor
  • address
  • area: area of the house(a reference id of area model)
  • metro: metro near the house(a reference id of metro model)
  • description_zh: description in Chinese
  • description_en: description in English
  • access_token: token after login
  • files: image of the house(send in request.files)

response: status 200 or 400

6. upload-image

upload pictures for house

url: /houses/upload-image/:house-id
variables:

  • house-id: the id of the house (required)
  • file: the picture file

response: status 200 or 400

7. search house

query the houses by some condition, return the data order by appointed field

method: REST (GET)
url: /houses/:lang/:orderBy/:page.:format
variables:

  • lang: language of returned (required, en|zh)
  • page: page of returned (required)
  • orderBy: the order of the response( format like "-publishDate" means order by publishDate desc, and "publishDate" means by publishDate asc)

  • format: format of returned (required, only support json currently)

    • keyword return the info which contains specified keyword in name_zh/name_en/description_zh/description_en/address
    • longitude
    • latitude
    • maxDistance(optional, default is 2(units is km))
    • price : format as 1-3(>=1 and <=3),5+(>=5)
    • bedrooms : format as 3(=3),1-4(>=1 and <=4),5+(>=5) (other params)
  • area: id of area (optional)
  • metro: id of metro (optional)
  • floor: floor of the house (optional)
  • houseType: type of the house (optional)
  • publisher: id of publisher (optional)
  • longitude： longitude of the house (optional)
  • latitude: latitude of the house (optional)
